# 勤怠編集ページ 詳細仕様

英語で考えて、日本語で説明してください。

スタッフ向け: `src/pages/attendance/edit/AttendanceEdit.tsx`
管理者向け: `src/pages/admin/AdminAttendanceEditor.tsx`

両ページは同じフォームロジックを共有し、スタッフは「修正理由」の入力が必要な点が異なる。

---

## 画面構成とフィールド一覧

### 定型入力

よく使う勤務パターンをワンクリックで複数フィールドに一括適用する機能。確認ダイアログを経由する。

| パターン | 動作 |
| --- | --- |
| クリア | 入力内容をすべてリセット |
| 通常勤務 | AppConfig の標準勤務時間・休憩時間を自動入力 |
| 定時出勤 | 出勤時間のみを標準時間で入力 |
| 定時退勤 | 退勤時間のみを標準時間で入力（必要に応じて昼休憩も設定） |

定型入力は**複数フィールドをまとめて更新する大きなアクション**。クイック入力との違いに注意。

### クイック入力チップ

勤務開始・終了フィールドの直下に表示される時刻ワンクリック入力ボタン。

- スタッフ・管理者で同一 UI
- クリックで即時 `setValue`（`shouldDirty: true`）→ 保存ボタンが活性化
- **単一フィールドのみ更新**（確認なし）。定型入力とは異なる

### 勤務日

表示のみ。対象日を示す。

### 勤務時間

#### 勤務開始時間
スタッフの勤務開始時刻。クイック入力ボタンで即時設定可能。

#### 勤務終了時間
スタッフの勤務終了時刻。クイック入力ボタンで即時設定可能。

#### 直行
スタッフが直接現場に向かうケース。ON にすると AppConfig の既定勤務開始時間が自動入力される。

#### 直帰
スタッフが直接帰宅するケース。ON にすると AppConfig の既定退勤時間が自動入力される。

### 休憩時間

開始・終了のペアで管理。「休憩時間を追加」ボタンで複数設定可能。昼休憩は自動設定のため手動追加不要。

#### 実労働時間（自動計算）
```
実労働時間 = 勤務終了時間 − 勤務開始時間 − 休憩時間合計
```
自動計算される表示専用値。

### 振替休日
特定日に勤務した代わりに別日を休日とする設定。日付を設定する際に 2 択が表示される：

- **クリアして設定**: 既存の勤務時間・休憩時間をリセットして振替休日として設定
- **クリアせず設定**: 既存データを保持したまま振替休日として設定

### 有給・特別休暇
選択すると勤務時間・休憩時間が AppConfig の既定値で自動入力される。

### 時間単位休暇
部分休暇。開始・終了時刻のペアで管理し、勤務時間内に収まる必要がある。勤務時間・休憩時間は自動調整される。

### 備考
特記事項・メモ用のテキストフィールド。遅刻・早退理由など。

### 修正理由（スタッフのみ）
スタッフが勤怠を修正する際に必須となる理由入力フィールド。管理者画面には存在しない。

## バリデーションルール

| ルール | 内容 |
| --- | --- |
| 終了 > 開始 | 勤務終了時間は勤務開始時間より後である必要がある |
| 休憩の重複禁止 | 複数の休憩時間が時間帯として重複してはならない |
| 法定休憩（6h 以上） | 最低 30 分の休憩確認を促す |
| 法定休憩（8h 以上） | 最低 1 時間の休憩確認を促す |
| 振替と有給の排他 | 振替休日は有給・特別休暇と同時設定不可 |
| 時間単位休暇の範囲 | 時間単位休暇の開始〜終了が勤務時間内に収まること |

## 技術的な実装メモ

- フォームは **React Hook Form + Zod** で管理する
- 時刻入力は `TimeInput` コンポーネント（`@shared/ui/`）を使う
- AppConfig 依存の既定値（標準勤務時間、昼休憩時間など）は `useAppConfig` フックで取得する
- バリデーションエラーは React Hook Form の `formState.errors` 経由で UI に表示する
