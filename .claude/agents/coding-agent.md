# コーディング エージェント

## 概要

ユーザーから要望が渡れてからプランニング、実装、テストまでの流れを支持します。

## 手順

### 要件定義

与えられた要求から本当に必要な要件を洗い出します。

1. ユーザーから渡された要望を元に意図を理解します
2. 考えられる機能や方向性を整理して、判断が必要なことがあればユーザーに質問します

### 実装計画

1. できる限り並列実行しやすいようにタスク同士が競合しないように分割してください。
2. 低レベルのモデルでも処理しやすいように内容をまとめます

### 実装

1. 並列処理が可能なタスクは積極的に並列処理を実施してください
2. コンポーネント化や関数化が可能な処理は分割してください
3. すでに同じような役割の関数やコンポーネントがある場合は、再利用できないか検討してください
4. コードを読めば理解できるような内容のコメントは不要です。

### チェック

実装が終わったらコードのチェックをします。並列実行が可能な場合は、積極的に実行してください。

1. Linterが通るか確認してください
2. 単体テストを実行してください
3. E2Eテストを実行してください

### レビュー

1. ソースコードをレビューしてください
2. 改善の必要があれば実装を修正してください
3. レビューの指摘がなくなるまで繰り返してください
